
## 理念概述

在“驾驭工程”框架中，内容创作不再是单向的“人类写作”或“AI生成”，而是一条精密的工业化流水线。 本框架的核心原则是：**人类提供意图、审美与判断（心力），Agent 提供发散、重组与格式化（算力）。**

为了实现完全的工程化，本工作流严格规定了**文件命名规范（版本号追踪）**、**数据流转方向**以及**独立技能（Agent Skills）的调用**。

## 核心数据流与命名规范

本工作流采用严格的数字前缀命名法，以确保上下文的不丢失和流程的可追溯：

- **`1.x` 阶段**：人类意图固化阶段（纯人类心力产出）。
    
- **`2.x` 阶段**：AI 架构与发散阶段（基于 1.x 的算力扩展）。
    
- **`3.x` 阶段**：收敛、叙事与定稿阶段（人机深度拉锯的最终文本）。
    

## 工作流详细拆解

### Step 1. 意图定调 (消化与构思)

**【阶段目标】** 将模糊的外部需求，转化为具有个人意志和初步解法的核心创意。本阶段完全由人类主导，Agent 处于待命状态。

- **输入：** 外部/甲方的原始需求（语音、聊天记录、简报等）。
    
- **人类动作：**
    
    1. **需求整理：** 消化原始需求，明确边界，手写整理出标准化的需求文档。
        
    2. **响应构思：** 思考“我该怎么做”，纯主观手写关于如何响应这个需求的项目初步创意。
        
- **Agent 动作：** 无（等待人类完成意图注入）。
    
- **标准输出：**
    
    - 📄 `1.1-Requirement` (需求文档)
        
    - 📄 `1.2-Ideation` (项目创意文档)
        

### Step 2. 框架分析 (系统骨架生成)

**【阶段目标】** 将人类粗颗粒度的“创意”，转化为系统化、结构化的“项目框架”。

- **输入：** 📄 `1.2-Ideation`
    
- **人类动作：** 向 Agent 下发设计指令，提供创意文档作为核心上下文。
    
- **Agent 动作：** * **调用 Skill：** `⚙️ theo-aiedu-analysis`
    
    - **执行逻辑：** 基于人类创意，进行结构化剖析，搭建课程/项目的骨架设计书。
        
- **标准输出：**
    
    - 📑 `2.1-framework` (项目架构设计书)
        

### Step 3. 专项研究 (模块化头脑风暴)

**【阶段目标】** 释放 Agent 算力，针对架构中的每一个节点进行穷尽式的内容发散、案例搜集与深度研究。

- **输入：** 📑 `2.1-framework`
    
- **人类动作：** 引导与监控，指令 Agent 严格按照架构书逐个模块展开研究。
    
- **Agent 动作：**
    
    - **调用 Skill：** `⚙️ theo-aiedu-brainstorm`
        
    - **执行逻辑：** 针对架构设计书中的每一个问题和知识点，全网检索、深度发散，生成海量的研究素材库。
        
- **标准输出：**
    
    - 📂 `2.2-brainstorm-fold` (专项研究子文档合集)
        

### Step 4. 收敛叙事 (驾驭循环核心区)

**【阶段目标】** 克服信息过载，将散乱的素材基于最初的意图强行收敛，并通过“评价-修改”的循环打磨出最终的讲课口语稿。这是整个工作流中最吃“心力”的一环。

- **联合输入：** 📄 `1.2-Ideation` + 📑 `2.1-framework` + 📂 `2.2-brainstorm-fold`
    
- **4.1 初稿生成：**
    
    - **Agent 动作：** 调用 Skill `⚙️ theo-aiedu-try`，融合所有输入上下文，强行收敛出讲课的基本叙事逻辑和初稿。
        
    - **中间输出：** 📄 `3.1-script-raft-v1.0`
        
- **4.2 驾驭循环 (Loop)：**
    
    - **人类动作：** 提供主观的严厉评价、逻辑审查（“这里不够抓人”、“案例太旧”、“语气太硬”）。
        
    - **Agent 动作：** 响应人类的反馈，持续修改、删减、迭代。
        
    - _说明：此过程将不断重复，直到人类认为达到了交付标准。_
        
- **标准输出：**
    
    - 🎯 `3.2-script-final` (确定的最终讲课口语稿)
        

### Step 5. 自动化交付 (全模态物料生成)

**【阶段目标】** 核心工作（文本与灵魂）已完成，将文本转化为多模态物理交付物的苦力活全部交给 AI。

- **输入：** 🎯 `3.2-script-final`
    
- **人类动作：** 验收最终成果，准备登台或发布。
    
- **Agent 动作：**
    
    - **调用对应 Skill 进行格式转化：**
        
        - `⚙️ theo-aiedu-output-ppt` ➔ 生成讲课 PPT 脚本/代码。
            
        - `⚙️ theo-aiedu-output-website` ➔ 生成配套演示网站 (HTML/React)。
            
        - `⚙️ theo-aiedu-output-card` ➔ 生成知识卡片或分发清单。
            
- **标准输出：**
    
    - 🚀 最终所有的实体交付物（PPT、网站、卡片等）。
        

## 结语

“驾驭”的本质，就是把人类从繁琐的“信息搬运”和“格式对齐”中解放出来，逼迫人类去面对最难、也是最有价值的问题：**我到底想表达什么？我该如何评判好坏？** 遵守本 SOP，你将拥有一个不知疲倦的高级智囊团和流水线执行团队。
